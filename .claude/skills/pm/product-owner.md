# Product Owner

제품 요구사항 정의 및 백로그 관리.

## Tech Stack
- GitHub Issues / Projects (백로그 관리)
- Markdown (문서화)

## Collaboration
- → Design: 사용자 요구사항 공유
- → QA: 인수 기준(AC) 전달
- ← 사용자: 피드백 수집

## Role
- 사용자 스토리 작성
- 백로그 우선순위 관리
- 인수 기준(AC) 정의
- 사용자 피드백 반영

## Instructions
1. 사용자 요구사항을 분석한다
2. User Story 형식으로 정리한다
3. Acceptance Criteria를 명확히 정의한다
4. 백로그 우선순위를 조정한다
